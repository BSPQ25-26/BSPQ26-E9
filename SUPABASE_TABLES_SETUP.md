# Supabase Setup: Schema, Roles and Tables

This document explains how to create the database structure for this project directly in Supabase SQL Editor.

## Scope
- One shared schema: `wallabot`
- Three DB roles:
  - `auth_service`
  - `inventory_service`
  - `transaction_service`
- Core tables based on current backend models:
  - `users`
  - `inventory_products`
  - `transaction_products`
  - `product_state_history`

## Important Context from This Repo
Current services map to these models:
- Auth service uses `users`
- Inventory service uses `inventory_products` with `seller_id` and `condition`
- Transaction service uses `transaction_products` and `product_state_history`

This is the domain-separated design (Option B): each service owns its own product table.

## How to Run in Supabase
1. Open Supabase project dashboard.
2. Go to `SQL Editor`.
3. Click `New query`.
4. Paste the script from the next section.
5. Replace role passwords if needed.
6. Click `Run`.

## Main SQL Script
```sql
begin;

-- 1) Shared schema
create schema if not exists wallabot;

-- 2) Roles (idempotent creation)
do $$
begin
  if not exists (select 1 from pg_roles where rolname = 'auth_service') then
    create role auth_service login password 'CHANGE_ME_AUTH';
  end if;

  if not exists (select 1 from pg_roles where rolname = 'inventory_service') then
    create role inventory_service login password 'CHANGE_ME_INVENTORY';
  end if;

  if not exists (select 1 from pg_roles where rolname = 'transaction_service') then
    create role transaction_service login password 'CHANGE_ME_TRANSACTION';
  end if;
end
$$;

-- Optional password rotation if roles already exist
-- alter role auth_service with password 'REAL_AUTH_PASSWORD';
-- alter role inventory_service with password 'REAL_INVENTORY_PASSWORD';
-- alter role transaction_service with password 'REAL_TRANSACTION_PASSWORD';

-- 3) Default schema per role
alter role auth_service set search_path = wallabot, public;
alter role inventory_service set search_path = wallabot, public;
alter role transaction_service set search_path = wallabot, public;

-- 4) Schema permissions
grant usage, create on schema wallabot to auth_service, inventory_service, transaction_service;

-- 5) users table (auth-service)
create table if not exists wallabot.users (
  id bigint generated by default as identity primary key,
  email text not null unique,
  password_hash text not null,
  wallet_balance double precision not null default 0.0,
  avg_rating double precision null,
  created_at timestamptz not null default now()
);

create index if not exists ix_users_email on wallabot.users(email);

-- 6) inventory_products table (inventory-service)
create table if not exists wallabot.inventory_products (
  id bigint generated by default as identity primary key,
  title varchar(255) not null,
  description text not null,
  category varchar(100) not null,
  price double precision not null check (price > 0),
  condition varchar(100) not null,
  state varchar(20) not null,
  seller_id varchar(255) not null,
  created_at timestamptz not null default now(),
  constraint ck_inventory_products_state
    check (state in ('Available','Reserved','Sold'))
);

create index if not exists ix_inventory_products_seller_id on wallabot.inventory_products(seller_id);
create index if not exists ix_inventory_products_state on wallabot.inventory_products(state);

-- 7) transaction_products table (transaction-service)
create table if not exists wallabot.transaction_products (
  id bigint generated by default as identity primary key,
  title varchar(255) not null,
  description varchar(1000) null,
  category varchar(100) null,
  price double precision not null check (price > 0),
  state varchar(20) not null,
  owner_id varchar(255) not null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint ck_transaction_products_state
    check (state in ('available','reserved','sold'))
);

create index if not exists ix_transaction_products_owner_id on wallabot.transaction_products(owner_id);
create index if not exists ix_transaction_products_state on wallabot.transaction_products(state);

-- 8) product_state_history table (transaction-service)
create table if not exists wallabot.product_state_history (
  id bigint generated by default as identity primary key,
  product_id bigint not null references wallabot.transaction_products(id) on delete cascade,
  from_state varchar(20) null,
  to_state varchar(20) not null,
  changed_at timestamptz not null default now(),
  changed_by varchar(255) null,
  constraint ck_history_states
    check (
      (from_state is null or from_state in ('available','reserved','sold'))
      and to_state in ('available','reserved','sold')
    )
);

create index if not exists ix_history_product_id on wallabot.product_state_history(product_id);
create index if not exists ix_history_changed_at on wallabot.product_state_history(changed_at);

-- 9) Table and sequence grants
grant select, insert, update, delete on all tables in schema wallabot
to auth_service, inventory_service, transaction_service;

grant usage, select, update on all sequences in schema wallabot
to auth_service, inventory_service, transaction_service;

alter default privileges in schema wallabot
grant select, insert, update, delete on tables
to auth_service, inventory_service, transaction_service;

alter default privileges in schema wallabot
grant usage, select, update on sequences
to auth_service, inventory_service, transaction_service;

commit;
```

## Verification Queries
Run these in SQL Editor after the main script.

### 1) Check tables in wallabot
```sql
select table_schema, table_name
from information_schema.tables
where table_schema = 'wallabot'
order by table_name;
```

### 2) Check columns and types
```sql
select table_name, column_name, data_type, is_nullable
from information_schema.columns
where table_schema = 'wallabot'
order by table_name, ordinal_position;
```

### 3) Check role search_path
```sql
select rolname, rolconfig
from pg_roles
where rolname in ('auth_service', 'inventory_service', 'transaction_service');
```

## Notes
- This script is safe to re-run (`if not exists` and role checks).
- If your service passwords were changed in Supabase, update `.env` accordingly.
- This script follows Option B and separates product tables by domain.

## Migration from legacy shared products table (optional)
If you already created `wallabot.products` from the old setup:

```sql
-- Legacy setup uses product_state_history -> wallabot.products.
-- Drop legacy tables first so Option B tables/FKs can be created cleanly.
drop table if exists wallabot.product_state_history cascade;
drop table if exists wallabot.products cascade;
```

Then run the main script again to create:
- `wallabot.inventory_products`
- `wallabot.transaction_products`
- `wallabot.product_state_history` (FK -> `transaction_products`)

If legacy `wallabot.products` contains data you need to keep, copy it before dropping and split rows manually into `inventory_products` and `transaction_products` (`seller_id` vs `owner_id`).
